## 🧪 测试步骤

### 1. 刷新浏览器
在 http://localhost:3000 按 **Ctrl + Shift + R** (硬刷新)

### 2. 检查控制台
按 **F12**, 切换到 **Console** 标签，应该看到：
```
[UI Patch] 主题切换补丁加载中...
[UI Patch] 主题切换补丁已成功加载 ✓
```

### 3. 视觉检查 - 总览模式

当前应该在总览模式，检查：
- [ ] 背景颜色是浅灰色 (不是白色)
- [ ] 两张卡片("基础版"和"AI 专家版")样式美观
- [ ] 卡片有阴影效果
- [ ] 鼠标悬停卡片时有上浮动画

### 4. 测试基础版模式

点击 **"基础版 (Basic)"** 卡片：
- [ ] 页面切换到工作界面
- [ ] 按钮颜色变为 **绿色**
- [ ] 文件上传区域边框悬停时变绿色
- [ ] 所有交互元素使用绿色主题

**在 Console 检查**：
```javascript
document.body.className  // 应显示包含 "mode-basic"
```

### 5. 测试专家版模式

点击返回按钮回到总览，然后点击 **"AI 专家版 (Expert)"**：
- [ ] 按钮颜色变为 **深蓝色**
- [ ] 所有交互元素使用蓝色主题
- [ ] 悬停效果正确

**在 Console 检查**：
```javascript
document.body.className  // 应显示包含 "mode-expert"
```

### 6. 功能测试

在任一模式下测试：
- [ ] 文件上传功能正常
- [ ] 文本处理功能正常
- [ ] 编辑器工作正常
- [ ] 所有按钮可点击
- [ ] 弹窗正常显示

### 7. 检查是否有错误

在 Console 标签：
- [ ] 无红色错误信息
- [ ] 无黄色警告（或只有预期的警告）

## ❌ 如果出现问题

### 样式没有应用
```javascript
// 在 Console 执行，检查加载的 CSS
Array.from(document.styleSheets).map(s => s.href)
// 应该看到 style-new.css
```

### 主题不切换
```javascript
// 检查补丁是否加载
Array.from(document.scripts).map(s => s.src).filter(s => s.includes('patch'))
// 应该看到 app-ui-patch.js
```

### 功能异常
立即回滚：
```bash
git restore public/index.html
```
然后刷新浏览器
