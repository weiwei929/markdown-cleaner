# 测试目录说明

本目录包含项目的测试资源和样例文件。

## 目录结构

- `samples/`: 包含用于测试 Markdown 处理功能的各种样例文件。
  - `test-document.md`: 综合测试文档，包含格式、标点、繁体等多种情况。
  - `quote-test.md`: 专门用于测试引号转换的文档。
  - 其他 `*-test.md` 文件：特定场景的测试文档。

## 如何使用

1. 启动应用：`npm start`
2. 打开浏览器访问 `http://localhost:3000`
3. 将 `samples/` 目录下的文件拖入上传区域
4. 观察处理结果是否符合预期

## 自动化测试

目前项目主要依赖手动测试。后续计划引入 Jest 进行单元测试。
